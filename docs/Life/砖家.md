## 领域
我现在涉足的技术领域比较杂，工作主要是DevOps，从微服务到SRE，从Java到Go到Python，从操作系统到计算机网络，从分布式系统到虚拟化技术，从编译到应用，都是浅尝辄止，接触不深，没有深入系统的学习与钻研。这些东西大致可以分为以下几个领域：    
- 软件工程：SRE 设计模式 微服务架构
- 编程语言：Java Go Python Shell  
- 框架：SpringBoot Django
- 计算机体系结构与理论：操作系统 计算机网络 分布式系统 虚拟化 编译原理   

具体来看，编程语言和框架是实践，在工作上，个人的能力最终通过编程实现；计算机体系结构与理论是指导实践的技术理论，是内功，不显山不露水，但是决定了能力的上下限；软件工程则是工程上的方法论，是更加抽象的东西。这只是我一个临时的分类，可能并不是非常科学。   
当前我主要负责机器管理与编译缓存两块工作，这两块工作的共同特点是——他们都不仅仅是一个微服务，还耦合了大量的机器运维、数据管理工作，除了微服务的Java代码，还需要维护shell脚本、go工具、docker镜像、VM镜像等，以及部分额外的附加的杂活。非主要的任务是编译加速、镜像缓存等任务。  
这些东西很难细分出具体的优先级，每样在日常工作中都需要用到，每一样我都是略懂。现在只能大概排一个学习顺序，明后天再具体细分一下计划。  
新接手的机器管理，需要快速学习的就是SRE，JAVA，分布式系统这几块的。5月份应该以IaC，Java的一堆框架，微服务源码，分布式锁这几块为主。搞定这个后，应该抽空看下设计模式与编译原理，下一步的工作重心可能是放在增量编译这一块。Django这一块，就看心情与空闲继续完善我的博客，没事翻译一下文档锻炼下英语。
